Dev Ops

Software infrastructure has greatly evolved over the years. This evolution has revolved around better management and orchestration.

In lay-man terms creating and managing servers has become a streamlined process with developers becoming a lot more involved in the process.

In the past developers wrote software and had to discuss the kind of infrastructure required to run it with server administrators. These days dev op engineers manage platforms that allow developers to describe the kind of infrastructure they will need in code.

Coding

You will need to be very good at scripting for the most part.

Working in teams

Dev Ops engineers can work solo or in a team with other dev op engineers.

Working alone

You can pick up this role in a consulting capacity and have no problems.

Impact

Dev Ops engineers manage the underlying infrastructure on top of which software solutions run. Importantly they make it possible for developers to just focus on the software solution they are working on.

Managing a team

You likely won't be managing a team.

Dealing with customers

You won't be interacting with the end customer.

Learning Curve

Besides learning scripting in a programming language, you will have to learn tools needed to work on particular cloud infrastructures and this can be hard for beginners who have never programmed beforehand.
