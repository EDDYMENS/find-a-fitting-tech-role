Information security Analyst

An information security analyst works to ensure that software products cannot easily be exploited through security loopholes.

Coding

Generally, the level of coding needed is the kind where you understand what it takes to build software solutions. You might require some scripting skills, this is useful for writing scripts to test out possible loopholes.

Working in teams

You will likely be interacting with developers learning as much as you can about the software product as well as providing them with information whenever you detect a possible threat.

Working alone

If you go the consulting route or if you work for a small company you are likely going to be spending most of your time working alone.

Impact

Information security analysts find themselves preventing hackers from accessing user data and preventing the misuse of software systems.

Managing a team

If you work in a large team with overlapping roles you might end up managing this team if you have a lot more experience within the team.

Dealing with customers

You will hardly have to deal with customers. On rare occasions, you might have to interact with an end-user if their data gets leaked through your system.

Learning Curve

There is a lot to learn in this field. You can start with scripting then move on to studying different tools of the trade. You will also need to keep up with new exploits as and when they are discovered.
